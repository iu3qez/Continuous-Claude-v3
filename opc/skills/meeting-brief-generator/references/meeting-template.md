# Meeting Brief Template

Complete template for structured meeting briefs with all sections.

## Full Template Structure

```markdown
# Meeting Brief: [Meeting Topic]

**Date:** [Date and time]
**Attendees:** [Names of participants]
**Project Context:** [Link to related Q4 goal project]
**Duration:** [Estimated meeting length]

## Quick Context
[2-3 sentence summary of meeting purpose, current state, and why this meeting is happening now]

## Key Topics
1. [Primary discussion topic with brief context]
2. [Secondary topic]
3. [Additional topics as needed]

## Preparation Checklist
- [ ] Review current project STATUS.md
- [ ] Gather relevant metrics or data
- [ ] Review open questions from last meeting
- [ ] Prepare specific examples or cases
- [ ] Check related task-master tasks

## Questions to Ask
1. [Strategic question about direction or priorities]
2. [Clarifying question about technical details]
3. [Timeline or resource question]
4. [Decision-making question]
5. [Next steps question]

## Expected Outcomes
- [Concrete outcome 1: Decision on X]
- [Concrete outcome 2: Agreement on Y]
- [Action items assigned with owners]
- [Timeline or milestones defined]

## Related Context
- **Project:** [Link to AI-Enablement-Q4-Goals/projects/XX-project-name/]
- **Status:** [Link to STATUS.md with current state]
- **Notes:** [Link to NOTES.md if relevant]
- **Tasks:** [Link to related .taskmaster tasks]
- **Previous Discussion:** [Link to prior meeting notes if applicable]

## Notes
[Space for additional context, constraints, or important background]
```

## Section Guidelines

### Quick Context
- Start with "why now" - what triggered this meeting
- Include current project state in 1 sentence
- State the primary goal or decision needed

### Key Topics
- Order by importance
- Include brief context for each
- Link to relevant documentation

### Preparation Checklist
- Make items specific and actionable
- Include time estimates where helpful
- Prioritize by impact on meeting success

### Questions to Ask
- Start with strategic/high-level
- Move to tactical/details
- End with next steps
- Frame as open questions, not yes/no

### Expected Outcomes
- Be specific about decisions needed
- Identify deliverables
- Note who needs to commit
- Set success criteria

### Related Context
- Use markdown links for easy navigation
- Include line numbers for specific sections
- Cross-reference with task-master tasks
- Link bidirectionally
